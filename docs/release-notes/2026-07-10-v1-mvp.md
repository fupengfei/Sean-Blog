## 需求说明

> 构建个人技术博客平台 Sean's AI World，包含前台展示（首页、博客、Skill 目录、项目、关于我）和 Admin 管理后台，为后续 AI 相关内容集成打下基础。

### 核心功能

1. **首页** — Hero 区 + 精选项目 + 精选文章 + CTA 底部栏（邮件/简历请求）
2. **博客模块** — 文章列表（分类/标签筛选 + 分页）+ 文章详情（MD 渲染 + 阅读计数）
3. **Skill 目录浏览** — 文件树浏览（左树右内容），支持 MD / 代码 / 图片预览
4. **项目展示** — 卡片网格展示项目列表，支持外部跳转
5. **关于我** — 个人介绍、技术栈、联系方式
6. **Admin 管理后台** — 文章管理、项目管理、文件目录管理、联系记录、分类/标签管理、修改密码
7. **JWT 认证** — 登录保护 Admin 接口，公开接口无需认证

### 决策：架构方案

**选择模块化单体架构。** 理由：v1 阶段业务简单、数据量小，单体架构降低部署和运维复杂度。模块按 auth/blog/project/file/contact 分包，代码边界清晰，后续可拆分为微服务。

## 思考过程

- **决策 1：Spring Boot 4.x 模块化单体** — 选择按业务模块分包而非按技术层分包，每个模块包含自己的 controller/service/entity/mapper，模块间通过 Service 层调用而非直接访问 Mapper，保证模块边界清晰 — trade-off：初期代码组织稍复杂，但后续拆分微服务成本极低

- **决策 2：MD 文件上传而非在线编辑器** — 技术博客内容通常用本地编辑器（VS Code / Obsidian）编写，上传 .md 文件比在线编辑器更符合实际工作流，后端自动解析 MD 为 HTML — trade-off：放弃了在线编辑的便捷性，换取了更专业的写作体验

- **决策 3：Docker Volume 挂载宿主机目录存储文件** — 文章 MD、Skill zip 解压文件、图片直接存储在宿主机 `/data/` 目录 — trade-off：避免了对象存储（S3/OSS）的额外成本和复杂度，但牺牲了水平扩展能力（v1 单机部署可接受）

- **决策 4：软删除代替物理删除** — 文章删除设置 `status = DELETED` 而非物理删除行，避免误删数据丢失 — trade-off：查询需要额外过滤条件，数据量增大后需要考虑清理策略

- **决策 5：JWT 无状态认证** — 选择 JWT 而非 Session，Admin 接口通过 `Authorization: Bearer <token>` 认证 — trade-off：无法主动失效 token（除非加黑名单），但避免了服务端会话存储，简化了部署

- **决策 6：前端 Next.js 14 App Router** — 选择 App Router 而非 Pages Router，利用 Server Components 减少客户端 JS 体积 — trade-off：学习曲线略高，但面向未来

### 关键 trade-off

| 决策 | 分类 | 理由 |
|------|------|------|
| 模块化单体 vs 微服务 | 复杂度 | v1 阶段业务简单，单体降低部署运维成本；模块边界清晰，后续拆分成本低 |
| MD 上传 vs 在线编辑器 | 用户体验 | 技术博客更适合本地编辑器写作流程，放弃在线编辑换取专业体验 |
| 宿主机文件存储 vs 对象存储 | 成本/复杂度 | v1 单机部署，避免云服务额外成本和复杂度；后续可迁移至 OSS |
| 软删除 vs 物理删除 | 数据安全 | 防止误删，以查询复杂度换取数据安全性 |
| JWT vs Session | 架构简洁性 | 无状态认证简化部署，放弃 token 即时失效能力 |
| Next.js 14 App Router vs Pages Router | 技术前瞻性 | 选择最新稳定方案，面向未来 |

## 已知限制

- **无自动化测试** — v1 优先功能实现，单元测试和集成测试将在后续迭代补充
- **无暗色模式** — CSS 变量体系已预留，但本期不做暗色切换
- **无评论系统** — 文章不支持评论，后续可集成 Giscus 等第三方方案
- **无全文搜索** — 仅支持标题/excerpt 关键词 LIKE 搜索，后续可升级为 Elasticsearch
- **无 RSS 订阅** — 后续迭代补充
- **无多用户/RBAC** — 仅支持单管理员，通过数据库初始化脚本创建
- **Admin 无访问统计面板** — Dashboard 仅有总数统计，无访问趋势图表
- **文件预览限制** — 超过 200KB 的文件不支持在线预览
- **图片上传无压缩** — 封面图直接存储原图，后续可加压缩/裁剪处理
- **Token 无主动失效机制** — JWT 签发后 24 小时内有效，无法手动踢出登录
- **单机部署** — 文件存储依赖本地磁盘，不支持水平扩展
